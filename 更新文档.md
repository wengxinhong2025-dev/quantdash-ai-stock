## 1. 总体结论

这轮更新的核心目标不是改 UI 风格，而是：

- 把超大组件和超大服务拆开，降低维护成本
- 把浏览器直连外部数据源改成 Python 可控代理层
- 给页面补齐“数据来源 / 更新时间 / 快照回退”信息
- 收紧鉴权、同步、数据源切换这些全局能力
- 把高频接口、异常处理、连接池、缓存、事务这些基础能力统一下沉到共享层
- 为第二数据源 `mootdx` 预留稳定接入点，且默认不影响现有主流程

当前结果：

- 页面主样式语言保持不变
- 后端已按业务域继续拆分
- 高频 EastMoney 请求已切到 Python 层
- 共享缓存、线程池、连接池、统一异常处理已落地
- 前端构建、类型检查、Lint、后端编译校验均已通过

## 9. 网络诊断结果

为定位 EastMoney 连接异常，已加入网络诊断脚本并生成结果：

- `scripts/diagnose_eastmoney_network.py`
- `data/system/eastmoney_network_diagnostics.json`

当前诊断结论：

- 这台机器访问 EastMoney 时，DNS/链路层存在异常
- 曾解析到 `198.18.0.x` 保留网段
- Python 层 `trust_env=True/False` 都可能得到 `ConnectError`

这说明：

- 代码层已基本收住浏览器直连问题和请求风暴
- 但如果本机网络/代理/网关拦截上游，仍会触发主源不可用
- 这也是第二数据源和快照回退需要保留的原因
